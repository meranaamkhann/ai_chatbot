Three checks run before a single reply is written

Every message passes through this before {{ brand_name }} is allowed to answer it.

01

Topic guard

A fast keyword pass catches the obvious cases instantly. Anything ambiguous or adversarially phrased ("ignore your instructions and…") falls back to a dedicated classifier call before the main model ever sees it. Off-topic requests get a polite redirect, not a generated answer.

02

Emergency detection

Language suggesting a medical emergency — chest pain, loss of consciousness, suicidal thoughts, severe bleeding — short-circuits straight to a fixed safety message pointing to emergency services. No model generation, no delay, no ambiguity.

03

Encrypted, account-scoped storage

Conversations are saved to your account so they're there next time you log in — but message content is encrypted at rest, never used for training, and "Clear all chats" deletes it immediately, for good.

Before you ask

Does it diagnose or prescribe anything?
No. It gives general, educational information and consistently points to a licensed clinician for anything specific to you, including medication dosages.
Is my conversation stored anywhere?
Yes — it's saved to your account, encrypted, so it's there next time you log in. It's never used for training, and "Clear all chats" deletes it immediately and permanently.
What happens if I ask something unrelated?
You'll get a short, friendly redirect back to health topics instead of an answer — that's the topic guard working as intended, not a bug.
Is this free to run myself?
Yes — it's built specifically to cost $0. Free Gemini API key, free hosting tier, no other services required. The source is linked below.
